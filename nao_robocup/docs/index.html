Nao Robocup simulation

Olivier Michel, Cyberbotics Ltd.
November 16th, 2007

Definitions

  • Robocup is an international competition of soccer playing robots: www.robocup.org
  • The Standard Platform League is a Robocup category in which all the robots are the same and operate fully autonomously, so that the competitors can focus on software development only: www.tzi.de/spl
  • Nao is a humanoid robot used in the Standard Platform League since 2008 and developed by Aldebaran Robotics: www.aldebaran-robotics.com
  • Webots is a fast prototyping and simulation software for mobile robots developed by Cyberbotics: www.cyberbotics.com
  • URBI is a programming language for mobile robots developed by Gostai: www.gostai.com

Preamble

Cyberbotics has developed a Webots simulation model including Nao robots and a soccer field used in the Standard Platform League. This model also includes a supervisor system allowing to fully simulate a complete match of the Standard Platform League. The model is distributed free of charge to the roboticist community and can be used with the free demo version of Webots. It allows users to develop the complete control software for the Standard Platform League and test it in simulation.

This model is intended to the following usages:
  • development tool for the competitors of the Standard Platform League.
  • development tool for the users of the Nao robot.
  • setup of a simulation-based competition ressembling the Standard Platform League.

Robot Model

The current version of the robot model includes the simulation of the following devices:
  • 22 servo motors with position control, speed control, force control, position feedback and force feedback.
  • 1 color LED in the logo displayed on the body of the robot.
  • 1 color camera with a resolution of 160x120 pixel and a horizontal field of view of 45 degrees.
  • 2 ultrasound distance sensors with a range of 70 cm.
  • 1 3D accelerometer in the body of the robot.
  • 8 pressure sensors (4 below each of the two feet of the robot).
  • 1 emitter device allowing the robots to send messages to their teammates.
  • 1 receiver device allowing the robots to receive messages from their their teammates.
Note: removing the pressure sensors makes the physics simulation run twice as fast.

Programming Languages

The Nao robot can be programmed using the following different programming languages:
  • C/C++: a sample program is provided.
  • URBI: a sample URBI program will be provided soon. TODO
  • Java: see Webots documentation.
  • Any third party program (Matlab, LabView, Lisp, etc.) through a TCP/IP connection.

Requirements

A physics simulation of two teams of 4 Nao robots (i.e., 8 robots) including intensive OpenGL camera usage (8 robot cameras and 1 main viewpoint camera) is a computer expensive process. Hence a powerful computer is highly recommended to run this simulation. We recommend the following hardware configuration:
  • an Intel Core 2 Duo, Intel Core 2 Quad, AMD64 X2 or better processor.
  • a recent nVidia or ATI graphics card with at least 128MB of RAM.
Since Webots runs on multiple platforms, you can choose either of these operating systems:
  • Linux 32 or 64 bit (any recent distribution)
  • Windows XP or Vista
  • Mac OS X 10.4 or 10.5
